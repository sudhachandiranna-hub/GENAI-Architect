Basics of Generative AI — LLMs, Open Source & API Keys

Before building anything with GenAI, we need the foundation: what LLMs are, open source vs closed models, when to use an API key vs running locally, and our first hands-on with Ollama.

6-Week Journey Week 1 · Day 5 of 6W · 5/45 (11.11%)

Today's Core Topic — Prerequisites of GenAI

01
Learn LLMs First
Before going into system building, we need to understand LLMs — both Open Source and Closed models — and where each is used.
02
Open Source vs Closed Models
Closed models (like OpenAI) need a purchased API key. Open source models can run locally on your own machine — no key required.
03
Hardware Needed
To run open source models locally, we need around 6–8GB of NVIDIA graphics (VRAM) for a smooth workflow.
💳 OpenAI API Key — Cost & Usage

After purchasing an OpenAI API key, it's pay & use — cost is generally around $10, starting from as little as $5. Free models are sometimes time-consuming in comparison. We need to buy an OpenAI API key today to move forward.

Hugging Face — The AI Developer's Social Network

🤗
Hugging Face = "Social Media for AI Developers"
Many companies test their applications here. 2 million+ open source models are available — it's a complete overview hub for the open source AI world.
📊 Choosing a Model — LLM Benchmark

We choose a model based on its LLM Benchmark score. For this course, we'll start with the lowest-spec option: Qwen 0.5B — small enough to run smoothly with 6–8GB VRAM.

Hands-On — Ollama

⚠ Free API Providers Have Limitations

Groq, NVIDIA, and OpenRouter all offer free or cheaper access — but each comes with usage limitations. Know these limits before building anything that depends on them at scale.

Thiru's Doubt-Clearing Technique

When you hit an error: paste the error and ask why it happened. Understand the cause, then fix it. This approach gives a much better understanding than just copying a fix without knowing why it worked.

Make More Mistakes — Make New Ones

Without mistakes, there is no learning — even the smaller ones matter. Keep making new mistakes rather than avoiding them; that's where the real understanding comes from.

— Thiru, continuing the mistakes philosophy from earlier days
